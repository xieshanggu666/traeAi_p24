# 漂流瓶皮肤功能修复说明

## 修复日期
2026-06-15

---

## 第一轮修复

### 问题1：皮肤页面中间皮肤未正常显示

**问题描述**：皮肤页面中"心有所属"到"绿意盎然"之间的皮肤没有正常显示出来。

**根本原因**：数据库中`bottle_skins`表存在旧的不完整数据，这些旧数据缺少`emoji`、`gradient_from`、`gradient_to`、`border_color`、`rarity`等关键字段，导致前端无法正常渲染皮肤卡片。

**修复方案**：修改`server/config/migrateSkins.js`中的皮肤数据插入逻辑，将`INSERT IGNORE`改为先查后更新/插入。

**修复位置**：`server/config/migrateSkins.js:211-242`

### 问题2：购买皮肤报错 Unknown column 'duration'

**根本原因**：数据库中`user_skins`表已存在但为旧版本表结构，缺少`duration`、`price`、`expire_at`、`is_active`等必需字段。`CREATE TABLE IF NOT EXISTS`在表已存在时不会自动添加缺失字段。

**修复方案**：在`server/config/migrateSkins.js`中添加`user_skins`表的字段修补逻辑，自动检测并添加缺失字段。

**修复位置**：`server/config/migrateSkins.js:176-194`

## 第二轮修复

### 问题3：皮肤列表返回15条数据，后面7条无样式

**问题描述**：商店皮肤接口返回了15条皮肤数据，但只有前8条有完整样式，后面7条因缺少渐变色、表情等字段导致页面无法正常渲染。

**根本原因**：数据库中残留了之前版本的旧皮肤记录（如"心有所属"、"绿意盎然"等），这些记录的ID不在当前`SKINS`数组中。迁移脚本的更新逻辑仅处理SKINS数组中定义的8款皮肤，旧记录不会被更新也不会被删除。

**修复方案**：在`server/config/migrateSkins.js`的迁移脚本末尾，插入/更新完8款皮肤后，删除所有ID不在SKINS数组中的旧记录：

```javascript
const validSkinIds = SKINS.map(s => s.id);
const placeholders = validSkinIds.map(() => '?').join(',');
const [deleteResult] = await connection.execute(
  `DELETE FROM bottle_skins WHERE id NOT IN (${placeholders})`,
  validSkinIds
);
```

**修复位置**：`server/config/migrateSkins.js:244-255`

**验证结果**：重启后日志显示 `已清理 8 条旧皮肤记录`

### 问题4：购买皮肤后首页漂流瓶样式没有变化

**问题描述**：用户购买并激活皮肤后，首页的漂流瓶图标仍显示默认的🍾，没有任何皮肤效果。

**根本原因**：`client/src/views/Home.vue`中的瓶子图标是硬编码的`<span class="bottle-icon">🍾</span>`，没有获取用户当前使用的皮肤数据。

**修复方案**：在Home.vue中添加以下逻辑：

1. **引入API**：导入`getMySkins`接口
2. **获取皮肤数据**：在`onMounted`中调用`fetchMySkin()`获取用户激活的皮肤
3. **动态显示**：
   - 瓶子图标从硬编码`🍾`改为`{{ activeSkinEmoji }}`，根据皮肤动态显示
   - 瓶子动画区域添加皮肤专属光晕背景（`bottleAnimBg`）
   - 瓶子浮动动画添加皮肤边框色投影（`bottleFloatStyle`）
   - 瓶子下方显示当前皮肤名称标签（`skinLabel`）

**修复位置**：`client/src/views/Home.vue`
- 模板：第24-31行
- 脚本：第72、79、86-110、132-139行
- 样式：第215-238行
