# 排行榜排名问题修复记录

## 问题描述

用户存在排名时，排行榜页面仍然显示"未上榜"。

## 问题根因分析

### 根因1：MySQL COUNT(*) 返回字符串类型

MySQL 的 `COUNT(*)` 查询返回的结果是 **字符串类型**（如 `"0"`、`"5"`），而不是 JavaScript 中的 Number 类型。

当执行 `myRank = myRankResult[0].rank + 1` 时：
- 预期：数字 5 + 1 = 6
- 实际：字符串 `"5" + 1 = "51"`（字符串拼接）

导致前端 `typeof rank === 'number'` 的判断失败，即使有排名也显示"未上榜"。

### 根因2：前端数据类型判断过于严格

前端使用 `typeof rank === 'number'` 进行判断，无法兼容字符串类型的数字。

### 根因3：列表数据字段类型不统一

排行榜列表中的 coins、charm 字段也可能是字符串类型，存在潜在的显示和排序问题。

## 修复方案

### 一、后端修复（server/routes/user.js）

#### 1. 统一数字类型转换

使用 `Number()` 和 `parseInt()` 双重确保所有数值字段为数字类型：

- `coins: Number(row.coins)`
- `charm: Number(row.charm)`
- `myRank = parseInt(countRows[0].cnt, 10) + 1`

#### 2. 显式构造返回对象

不再直接返回数据库查询结果，而是手动构造返回对象，确保每个字段类型正确：

```javascript
list = rows.map(row => ({
  id: row.id,
  nickname: row.nickname,
  avatar: row.avatar,
  coins: Number(row.coins),
  charm: Number(row.charm)
}));
```

#### 3. myInfo 也做同样处理

```javascript
const myInfo = myInfoRows[0] ? {
  id: myInfoRows[0].id,
  nickname: myInfoRows[0].nickname,
  avatar: myInfoRows[0].avatar,
  coins: Number(myInfoRows[0].coins || 0),
  charm: Number(myInfoRows[0].charm || 0)
} : null;
```

#### 4. 增加错误日志

在排名计算失败时输出错误日志，方便排查问题：

```javascript
} catch (e) {
  console.error('计算财富排名出错:', e);
  myRank = null;
}
```

### 二、前端修复（client/src/views/Rank.vue）

#### 1. 移除计算属性，改用工具函数

使用 `formatRank()` 和 `hasRank()` 两个普通函数替代计算属性，代码更简洁，且每次调用都会重新计算：

```javascript
function formatRank(rank) {
  if (rank === null || rank === undefined || rank === '') return '未上榜';
  const num = parseInt(rank, 10);
  if (isNaN(num) || num <= 0) return '未上榜';
  return num;
}

function hasRank(rank) {
  if (rank === null || rank === undefined || rank === '') return false;
  const num = parseInt(rank, 10);
  return !isNaN(num) && num > 0;
}
```

#### 2. 显式解构赋值

数据赋值时显式解构，避免响应式丢失：

```javascript
wealthRank.value = {
  list: result.list || [],
  myRank: result.myRank,
  myInfo: result.myInfo || null
};
```

#### 3. 模板中调用函数

```html
<div class="my-rank-num" :class="{ 'no-rank': !hasRank(wealthRank.myRank) }">
  {{ formatRank(wealthRank.myRank) }}
</div>
```

#### 4. 移除调试日志

移除 `console.log` 调试语句，保持代码整洁。

1. 重启后端服务：`cd server && npm start`
2. 前端重新访问排行榜页面
3. 检查：
   - 数值为 0 的用户显示"未上榜"（灰色小字号）
   - 数值 > 0 的用户显示正确的排名数字（紫色大字号）
   - 排行榜列表按数值降序排列
   - 前三名显示奖牌图标

## 经验总结

1. **MySQL COUNT() 返回字符串**：这是一个常见的坑，所有 COUNT 查询结果都要用 `parseInt()` 或 `Number()` 转换
2. **前后端数据类型要统一**：接口返回的数据应确保类型正确，不要依赖前端做类型转换
3. **使用 parseInt 更安全**：相比 `typeof === 'number'`，`parseInt` 能兼容更多输入格式
4. **显式构造返回对象**：不要直接把数据库查询结果返回给前端，中间加一层转换更可控
