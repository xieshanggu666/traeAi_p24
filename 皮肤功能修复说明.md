# 漂流瓶皮肤功能修复说明

## 修复日期
2026-06-15

## 问题1：皮肤页面中间皮肤未正常显示

### 问题描述
皮肤页面中"心有所属"到"绿意盎然"之间的皮肤没有正常显示出来。

### 根本原因
数据库中`bottle_skins`表存在旧的不完整数据，这些旧数据缺少`emoji`、`gradient_from`、`gradient_to`、`border_color`、`rarity`等关键字段，导致前端无法正常渲染皮肤卡片。

### 修复方案
修改`server/config/migrateSkins.js`中的皮肤数据插入逻辑：
- 原逻辑：使用`INSERT IGNORE`仅插入不存在的皮肤，已存在的旧数据不会更新
- 新逻辑：先查询皮肤是否存在，存在则`UPDATE`所有字段为最新完整值，不存在则`INSERT`

### 修复位置
`server/config/migrateSkins.js:211-242`

---

## 问题2：购买皮肤报错：Unknown column 'duration' in 'field list'

### 问题描述
点击购买皮肤时提示"购买皮肤失败"，后端接口报错：`Error: Unknown column 'duration' in 'field list'`。

### 根本原因
数据库中`user_skins`表已存在但为旧版本表结构，缺少`duration`、`price`、`expire_at`、`is_active`等必需字段。虽然迁移脚本的`CREATE TABLE IF NOT EXISTS`定义了这些字段，但由于表已存在，不会自动添加缺失字段。

### 修复方案
在`server/config/migrateSkins.js`中添加`user_skins`表的字段修补逻辑（与`bottle_skins`表类似）：
1. 查询`INFORMATION_SCHEMA.COLUMNS`获取表中已存在的字段
2. 对比必需字段列表，自动`ALTER TABLE ADD COLUMN`添加缺失字段

### 修复位置
`server/config/migrateSkins.js:176-194`
